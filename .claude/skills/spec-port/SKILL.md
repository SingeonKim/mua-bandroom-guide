# spec-port — PA 스펙 문서 이식·갱신

작성자용 PA 세팅 원본을 MkDocs Material에서 잘 보이는 `docs/spec.md`로 옮기고, 이후 값 변경을 반영하는 방법.

## 왜 3단계 구분이 핵심인가

이 문서의 값에는 신뢰 수준이 다른 3종이 섞여 있다. 독자(관리자)가 "이 값을 믿고 써도 되는지" 즉시 판단할 수 있어야 한다. 그래서 시각적으로 구분한다.

| 단계 | 의미 | 표기 방법 |
|------|------|----------|
| **확정** | 실측·검증 완료 | 표/본문에 그대로 (별도 표시 없음) |
| **잠정** | 혼자 확인, 합주 재검증 필요 | `!!! warning "잠정 — 합주 재검증 필요"` 또는 셀에 `(잠정)` 유지 |
| **빈칸** | 현장에서 채울 것 | `______` 유지, 가능하면 `!!! note "현장에서 채울 것"`로 묶음 |

원본에서 "잠정", "______" 표기가 이미 단서다. 이를 놓치지 말고 그대로 분류 근거로 삼는다.

## 이식 절차

1. **원본을 통째로 읽는다.** 장(0~8 + 작업 순서) 구조를 그대로 유지한다.
2. **형식만 변환한다.** 내용·수치·주의사항은 보존한다.
   - 신호 흐름 다이어그램 → ```` ``` ```` 코드블록
   - ※/⚠️ 주의 → 본문 유지 또는 `!!! warning`/`!!! danger`
   - 표 → Markdown 표 그대로 (MkDocs `tables` 확장이 렌더링)
3. **3단계를 표시한다** (위 표 기준).
4. **확정 정보 목록을 뽑는다** — guide-writer에게 넘길, 이용자에게 공개 가능한 확정 사실만. (잠정/게인 수치 제외)

## 확정 정보 목록 작성 기준

guide로 전달할 수 있는 것 / 없는 것을 명확히 가른다.

**전달 가능(확정·안전):**
- 표준 악기 배치 ch 매핑 (베이스 ch8, 키보드 ch9~12, 기타 ch13/14, 보컬 ch17~19)
- 씬 불러오기 원칙("건드렸으면 씬 다시 불러오기")
- 캐비넷/스피커 안전수칙(튜브앰프 스피커선 빼지 말 것 등)
- "게인·라우팅·48V 건드리지 마라"는 운영 규칙

**전달 금지(잠정·위험):**
- 구체적 dB 게인값(베이스 +21dB, 케이스별 -10/0/+10 등)
- 모니터 분배 수치(-10/-15), 메인 페이더/본체 수치(잠정)
- 컴프/게이트/EQ 파라미터
- 빈칸(`______`) 항목 전체

## 갱신 모드 (재호출)

`docs/spec.md`가 이미 있으면 새로 쓰지 말고 갱신한다.
- **잠정→확정**: 해당 값의 `!!! warning`/`(잠정)`을 제거, 확정값으로 교체. guide-writer에 통보.
- **빈칸 채움**: `______`를 실값으로 교체, `!!! note` 묶음에서 제외.
- 변경 안 된 부분은 손대지 않는다.

## 주의

- 확정/잠정이 모호하면 **잠정으로 분류**(안전 우선)하고 오케스트레이터에 "확인 필요"로 올린다.
- PA 도메인 용어(Pre/Post-fader, Bus, 라인아웃 구조)의 의미를 바꾸지 않는다. 모르면 원본 표현 보존.
